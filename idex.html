Labo-lecture : gestion des fichiers et dossiers en PowerShell

Dans ce labo-lecture, vous allez reprendre les notions de la leçon et les exécuter vous-même dans un dossier de travail sécurisé. Le but est de manipuler les cmdlets de base pour créer, lister, renommer, copier, déplacer et supprimer des dossiers et des fichiers, puis d’aller plus loin avec la recherche, le filtrage, le tri, la détection de doublons, le rangement automatique par type et la consultation des permissions. Chaque commande doit être tapée sur votre propre machine, puis vérifiée par une capture lorsque c’est demandé. Les chemins proposés sont des exemples sûrs dans votre profil Windows. Adaptez seulement le nom d’utilisateur affiché dans le prompt si le vôtre est différent.

objectifs de la leçon

  • Créer un environnement de travail propre pour manipuler les fichiers et les dossiers sans risque.
  • Appliquer les cmdlets PowerShell vues dans la leçon pour gérer les répertoires et les fichiers.
  • Utiliser la recherche, le filtrage et le tri pour retrouver rapidement des éléments utiles.
  • Tester des scénarios avancés : doublons, rangement automatique et consultation des permissions.
  • Conserver des preuves visuelles claires et préparer un PDF final propre à remettre.

Partie 1 — préparation du dossier de travail

Consignes générales

Travaillez dans un dossier de labo dédié. N’exécutez pas les commandes de suppression sur vos vrais documents. Le prompt affiché dans les exemples utilise le nom anis uniquement comme modèle. Sur votre poste, le prompt peut afficher un autre nom d’utilisateur et un autre chemin : c’est normal.

Rappel : dans ce labo, l’ordre attendu est toujours : lire la consigne, taper la commande, vérifier le résultat, puis insérer la capture demandée.
Rappel : si vous utilisez PowerShell 7 ou Windows PowerShell 5.1, le prompt peut varier légèrement. Ce n’est pas un problème tant que la commande et son résultat sont corrects.

Partie 5 — détection de doublons

Créer, repérer et simuler la suppression des doublons

Dans cette partie, vous allez créer volontairement plusieurs fichiers identiques, calculer leur hash, regrouper les doublons et simuler leur suppression avec -WhatIf.

Commande à taper 
PS C:\Users\anis> $dup = "$env:USERPROFILE\Documents\PS_Cours7_Lab\04_Doublons"
Set-Content -Path "$dup\original.txt" -Value "même contenu"
Copy-Item "$dup\original.txt" "$dup\copie1.txt" -Force
Copy-Item "$dup\original.txt" "$dup\copie2.txt" -Force
$doublons = Get-ChildItem "$dup" -File -Recurse | Get-FileHash |
 Group-Object -Property Hash | Where-Object { $_.Count -gt 1 }
foreach($grp in $doublons) {
    Write-Host "Doublons trouvés (Hash $($grp.Name)) :"
    $grp.Group | ForEach-Object { Write-Host " $($_.Path)" }
}
Get-ChildItem "$dup" -File -Recurse | Get-FileHash |
 Group-Object Hash | Where-Object { $_.Count -gt 1 } |
 % { $_.Group } | Out-File -FilePath "$dup\Duplicates.txt"
Get-ChildItem "$dup" -File -Recurse | Get-FileHash |
 Group-Object Hash | Where-Object { $_.Count -gt 1 } |
 ForEach-Object {
    $_.Group | Select-Object -Skip 1 | Remove-Item -WhatIf
 }

Partie 6 — rangement automatique par type

Exécuter un script d’organisation automatique

Vous allez préparer un petit dossier désordonné, puis lancer un script PowerShell qui classe automatiquement les fichiers selon leur extension.

Préparer les fichiers à ranger

Commande à taper 
PS C:\Users\anis> $org = "$env:USERPROFILE\Documents\PS_Cours7_Lab\06_A_organiser"
New-Item -Path "$org\facture.pdf" -ItemType File -Force
New-Item -Path "$org\photo.jpg" -ItemType File -Force
New-Item -Path "$org\notes.txt" -ItemType File -Force
New-Item -Path "$org\rapport.docx" -ItemType File -Force

Lancer le script de rangement

Commande à taper 
PS C:\Users\anis> $TargetFolder = "$env:USERPROFILE\Documents\PS_Cours7_Lab\06_A_organiser"
Set-Location -Path $TargetFolder
$files = Get-ChildItem -File
foreach ($file in $files) {
    $ext = $file.Extension.TrimStart('.').ToLower()
    if ($ext -eq "") {
        continue
    }
    $destFolder = Join-Path $TargetFolder $ext
    if (!(Test-Path $destFolder)) {
        New-Item -Path $destFolder -ItemType Directory | Out-Null
    }
    Move-Item -Path $file.FullName -Destination $destFolder
}
Get-ChildItem -Path $TargetFolder -Recurse

Partie 7 — permissions et ACL

Consulter et recopier des ACL

Cette partie reprend la logique de la leçon sur Get-Acl et Set-Acl. Travaillez uniquement dans votre dossier de labo.

Préparer les éléments de test

Commande à taper 
PS C:\Users\anis> $aclPath = "$env:USERPROFILE\Documents\PS_Cours7_Lab\05_ACL"
New-Item -Path "$aclPath\Docs" -ItemType Directory -Force
New-Item -Path "$aclPath\Dog.txt" -ItemType File -Force
New-Item -Path "$aclPath\Cat.txt" -ItemType File -Force
Get-Acl -Path "$aclPath\Docs" | Format-List

Copier les permissions d’un fichier vers un autre

Commande à taper 
PS C:\Users\anis> $aclDog = Get-Acl -Path "$aclPath\Dog.txt"
Set-Acl -Path "$aclPath\Cat.txt" -AclObject $aclDog
(Get-Acl "$aclPath\Cat.txt").Access | ForEach-Object {
    $_ | Format-List IdentityReference, FileSystemRights, AccessControlType
}

Ajouter une règle d’accès sur votre propre dossier

Commande à taper 
PS C:\Users\anis> $acl = Get-Acl "$aclPath\Docs"
$utilisateur = "$env:USERDOMAIN\$env:USERNAME"
$droit = [System.Security.AccessControl.FileSystemRights]::ReadAndExecute
$type = [System.Security.AccessControl.AccessControlType]::Allow
$nouvelleRegle = New-Object System.Security.AccessControl.FileSystemAccessRule($utilisateur, $droit, $type)
$acl.AddAccessRule($nouvelleRegle)
Set-Acl -Path "$aclPath\Docs" -AclObject $acl
(Get-Acl "$aclPath\Docs").Access | ForEach-Object {
    $_ | Format-List IdentityReference, FileSystemRights, AccessControlType
}
Rappel : si votre poste refuse cette dernière modification, gardez tout de même une capture du message affiché puis passez à la partie suivante. La consultation et la copie d’ACL restent les deux opérations minimales obligatoires.
